# 静态障碍物距离阈值预警效果测试 — 操作手册

> 本文档记录从人工标注到指标统计的完整流程，涉及脚本、数据格式与使用命令。
> 适用场景：仅具备人工等级真值（无距离真值），聚焦算法预警正确程度评估。
> **v1.1 更新**：新增时间戳对齐模式，支持录屏视频与 ROS bag 异频数据比对，支持时间偏移与倍速修正。

---

## 一、整体流程概览

```
视频文件
   │
   ▼
人工标注（片段式，4 个等级）
   │  └─ 输出: ground_truth.csv（含帧区间 + 时间区间）
   ▼
系统预测导出（逐帧/逐时间点）
   │  └─ 输出: prediction.csv（帧格式 或 时间戳格式）
   ▼
指标统计脚本自动计算（自动识别对齐模式）
   │  └─ 输出: 控制台报告 + LaTeX 表格代码
   ▼
直接贴入论文 D4 表
```

---

## 二、核心文件清单

| 文件 | 路径 | 作用 |
|------|------|------|
| 标注脚本 | `tools/annotate_video.py` | 边看视频边按键标注，导出片段式 CSV（同时含帧和时间） |
| 统计脚本 | `tools/eval_static_warning.py` | 读取真值 + 预测，自动识别对齐模式，计算全部指标 |
| 导出脚本 | `tools/export_bag_prediction.py` | 从 ROS bag 提取预警等级 topic，导出预测 CSV |
| 本手册 | `tools/静态预警效果测试_操作手册.md` | 即本文档 |

## 三、标注阶段

### 3.1 环境准备

```bash
pip install opencv-python
```

### 3.2 启动标注

```bash
python tools/annotate_video.py \
    --video "/path/to/your_video.mp4" \
    --output "ground_truth.csv"
```

### 3.3 按键说明

| 按键 | 功能 |
|------|------|
| `1` | 标记当前帧开始为「无危险」 |
| `2` | 标记当前帧开始为「提示」 |
| `3` | 标记当前帧开始为「报警」 |
| `4` | 标记当前帧开始为「紧急」 |
| `空格` | 暂停 / 继续 |
| `← / →` | 暂停时逐帧进退（找精确切换点） |
| `q` | 保存并退出 |

### 3.4 标注技巧

- 视频默认从「无危险」开始，无需按 `1`。
- 看到等级切换时，先 `空格` 暂停，再用 `→` 找到精确切换帧，然后按对应数字键。
- 脚本会自动记录上一个片段的结束帧（当前帧 - 1），无需手动填区间。
- 标注完成后，控制台会打印所有片段区间，供复核。

### 3.5 输出格式（ground_truth.csv）

```csv
video_name,start_frame,end_frame,level,start_time_sec,end_time_sec
scene1,0,150,无危险,0.000,5.000
scene1,151,210,提示,5.033,7.000
scene1,211,260,报警,7.033,8.667
scene1,261,340,紧急,8.700,11.333
```

**字段说明：**

- `video_name`：场景/视频名称。多个场景可放在同一个文件，也可分文件。
- `start_frame`, `end_frame`：闭区间，包含首尾两帧。
- `level`：只能是以下四个字符串之一，**严格区分大小写**：`无危险`、`提示`、`报警`、`紧急`。
- `start_time_sec`, `end_time_sec`：基于 `frame / fps` 自动计算的视频播放时间（秒），**用于与 ROS 等异频数据对齐**。

## 四、系统预测导出

### 4.1 从 ROS bag 导出（推荐）

使用提供的导出脚本：

```bash
python tools/export_bag_prediction.py \
    --bag /path/to/your.bag \
    --topic /warning_level \
    --video-name "scene1" \
    --output prediction.csv
```

脚本会提取指定 topic 的预警等级消息，输出带 `timestamp_sec` 的 CSV。

**导出前必须修改 `export_bag_prediction.py` 中的 `extract_level` 函数**，以匹配你的 ROS 消息格式。常见情况：

- `msg.level` 是整型 0~3：用 `LEVEL_MAP` 映射
- `msg.level` 是字符串（如 `"报警"`）：直接返回
- `msg.data` 是字符串（`std_msgs/String`）：直接返回

### 4.2 手动构造 prediction.csv

如果已有其他来源的系统输出，需组织为以下两种格式之一：

**格式 A（时间戳对齐，推荐用于录屏 vs ROS 异频场景）：**

```csv
video_name,timestamp_sec,level
scene1,0.00,无危险
scene1,0.10,无危险
scene1,0.20,提示
...
```

**格式 B（帧对齐，仅当视频与系统输出帧率完全一致时使用）：**

```csv
video_name,frame_id,level
scene1,0,无危险
scene1,1,无危险
...
```

**关键约束：**

- `video_name` 必须与真值文件中的名称一致，否则无法对齐。
- `timestamp_sec` 是相对于场景起点的连续时间（秒），从 0 开始递增。
- `level` 同样只能是 `无危险` / `提示` / `报警` / `紧急`。
- 文件必须包含表头。

## 五、指标统计

### 5.1 运行命令

**基础用法（帧对齐模式，预测含 `frame_id`）：**

```bash
python tools/eval_static_warning.py \
    --gt ground_truth.csv \
    --pred prediction.csv
```

**时间对齐模式（预测含 `timestamp_sec`，自动识别）：**

```bash
python tools/eval_static_warning.py \
    --gt ground_truth.csv \
    --pred prediction.csv
```

**带时间偏移（录屏比 ROS 晚开始）：**

```bash
python tools/eval_static_warning.py \
    --gt ground_truth.csv \
    --pred prediction.csv \
    --time-offset 12.0
```

**带时间偏移 + 倍速修正（录屏时以 0.5 倍速播放 rosbag）：**

```bash
python tools/eval_static_warning.py \
    --gt ground_truth.csv \
    --pred prediction.csv \
    --time-offset 9.154 \
    --time-scale 0.5
```

### 5.2 对齐模式说明

评估脚本自动检测预测文件格式，选择对齐模式：

| 预测文件列名 | 对齐模式 | 适用场景 |
|------|------|----------|
| `timestamp_sec` | **时间对齐** | 录屏视频与 ROS bag 帧率不同；视频与系统输出不同步 |
| `frame_id` | 帧对齐 | 视频与系统输出帧率完全一致且同步 |

**时间对齐的映射公式：**

评估脚本将真值区间（视频时间）按以下公式映射到预测时间轴：

```
rosbag_time = time_offset + video_time * time_scale
```

- `time_offset`：视频开始时刻相对于 ROS bag 起点的延迟（秒）。视频比 ROS 晚开始时设正值。
- `time_scale`：录屏播放倍率。以 0.5 倍速播放则设 0.5，原速设 1.0。

**示例：** 视频从 rosbag 第 12s 开始，且以 0.5 倍速播放：
- 视频时间 0s → rosbag 时间 `12 + 0 * 0.5 = 12.0s`
- 视频时间 1s → rosbag 时间 `12 + 1 * 0.5 = 12.5s`
- 视频时间 10s → rosbag 时间 `12 + 10 * 0.5 = 17.0s`

### 5.3 支持的输入格式

**真值文件（`--gt`）**支持自动识别三种格式：

| 格式 | 必需列 | 适用场景 |
|------|--------|----------|
| 片段式帧+时间（推荐） | `video_name,start_frame,end_frame,level,start_time_sec,end_time_sec` | 新版 `annotate_video.py` 产出 |
| 片段式仅时间 | `video_name,start_time_sec,end_time_sec,level` | 手动标注时间区间 |
| 逐帧式 | `video_name,frame_id,level` | 程序自动生成真值时 |

**预测文件（`--pred`）**支持两种格式：

| 格式 | 必需列 | 对齐模式 |
|------|--------|----------|
| 时间戳格式（推荐） | `video_name,timestamp_sec,level` | 时间对齐 |
| 逐帧格式 | `video_name,frame_id,level` | 帧对齐 |

### 5.4 输出指标说明

脚本会输出两类结果：控制台报告 + LaTeX 表格代码。

#### 控制台报告内容

| 指标 | 符号 | 含义 |
|------|------|------|
| **混淆矩阵** | — | 真值等级 × 预测等级的交叉计数 |
| **精确率（按等级）** | $P_i$ | 系统判定为等级 $i$ 时，实际也为 $i$ 的比例 |
| **召回率（按等级）** | $R_i$ | 实际为等级 $i$ 时，系统成功判为 $i$ 的比例 |
| **F1 分数（按等级）** | $F1_i$ | 精确率与召回率的调和平均 |
| **分级准确率** | Acc | 预测等级与真值等级完全一致的帧占比 |
| **误报率（过度预警率）** | FAR | 预测等级 **高于** 真值等级的帧占比 |
| **漏报率（预警不足率）** | MDR | 预测等级 **低于** 真值等级的帧占比 |
| **平均绝对误差** | MAE | 等级编码（0~3）后，\|预测 - 真值\| 的平均值 |

#### LaTeX 输出内容

脚本自动生成两张表的 LaTeX 代码：

1. **表 D4a**：分级有效性统计表（精确率 / 召回率 / F1 / 准确率 / 误报率 / 漏报率 / MAE）
2. **表 D4b**：等级混淆矩阵

可直接复制粘贴到论文 `chap05.tex` 的对应位置。

## 六、指标解读指南

### 6.1 误报率 vs 漏报率的工程含义

| 指标 | 工程含义 | 风险 |
|------|----------|------|
| **误报率（FAR）** | 系统太敏感，"狼来了" | 操作员产生预警疲劳，可能忽视真实危险 |
| **漏报率（MDR）** | 系统太迟钝，"该报不报" | 错过真实危险，直接导致安全事故 |

**在吊装安全场景中，漏报比误报更致命。** 因此理想状态是：MDR 尽量接近 0，FAR 在可接受范围内。

### 6.2 MAE 的额外价值

准确率（Acc）把「紧急→报警」和「紧急→无危险」视为同等错误，这不合理。MAE 能区分错误严重程度：

- 「紧急→报警」：MAE = 1
- 「紧急→无危险」：MAE = 3

**MAE 越小，说明错误集中在相邻等级之间，系统总体偏保守但无严重漏报。**

### 6.3 混淆矩阵的典型模式

| 模式 | 表现 | 诊断 |
|------|------|------|
| 上三角密集 | 预测等级普遍高于真值 | 系统过度预警，阈值偏保守 |
| 下三角密集 | 预测等级普遍低于真值 | 系统预警不足，阈值偏宽松 |
| 副对角线密集 | 相邻等级误判多 | 阈值边界附近抖动，可考虑加大迟滞 |
| 远离对角线有值 | 出现跳级（如无危险→紧急） | 状态机或输入存在异常跳变 |

## 七、常见问题

### Q1：真值和预测的帧数/时间对不上？

**帧对齐模式**：脚本只统计两者**交集**中的帧。如果某帧只在真值中出现、但预测中没有，该帧会被忽略。

**时间对齐模式**：脚本对预测的每个 `timestamp_sec`，在真值的时间区间中查找对应等级。不在任何真值区间内的预测时间点会被忽略。

**建议**：确保两个文件覆盖同一时间段，且 `video_name` 完全一致。

### Q2：录屏与 ROS bag 不同步（延迟 + 倍速）？

使用 `--time-offset` 和 `--time-scale` 参数。映射公式：

```
rosbag_time = time_offset + video_time * time_scale
```

**常见场景对照：**

| 场景 | `--time-offset` | `--time-scale` |
|------|-----------------|----------------|
| 同时开始，原速播放 | `0` | `1.0` |
| 视频晚 12s 开始，原速 | `12` | `1.0` |
| 同时开始，0.5 倍速播放 | `0` | `0.5` |
| 视频晚 9.154s 开始，0.5 倍速 | `9.154` | `0.5` |

### Q3：视频太长（如 20 分钟），人工标注工作量太大？

两种策略：

1. **全量标注**：如果视频确实包含多次完整的等级变化，且你需要整体统计，那就完整标。可以分段完成，保存多个 CSV 后合并。
2. **抽样标注**：如果视频大部分时间停留在同一等级，只抽取**等级切换边界前后各 5~10 秒**的关键片段单独标注。此时需在论文中注明评价基于抽样帧。

### Q4：标注脚本在 macOS 上窗口闪退？

OpenCV 的 `cv2.imshow` 在部分 macOS 环境需要添加以下代码到脚本头部：

```python
import os
os.environ['QT_QPA_PLATFORM'] = 'xcb'  # 或 'offscreen'
```

如果仍有问题，改用**方案二（Excel + 视频播放器）**手动标注。

### Q5：系统输出的等级名称和脚本要求不一致？

统一替换为四个标准字符串：`无危险`、`提示`、`报警`、`紧急`。可在预测 CSV 生成阶段做映射，例如：

```python
level_map = {"safe": "无危险", "warn": "提示", "alert": "报警", "danger": "紧急"}
```

### Q6：如何确定 `time_offset` 的具体数值？

如果录屏和 ROS 不是严格同步开始的，可以通过对比两者总时长粗略估算：

```python
import cv2
cap = cv2.VideoCapture("your_video.mp4")
fps = cap.get(cv2.CAP_PROP_FPS)
total_frames = int(cap.get(cv2.CAP_PROP_FRAME_COUNT))
video_duration = total_frames / fps
print(f"视频时长: {video_duration:.1f}s")
```

若 ROS bag 时长比视频长约 `X` 秒，则 `time_offset` 约为 `X`。更精确的做法是在 ROS 和视频中各找一个明确的事件（如吊钩开始移动），手动计算两者时间差。

## 八、完整工作流示例

假设你的视频是 `/Users/cyf/Movies/塔吊标准节-综合预警-评估.mp4`，视频从 rosbag 第 9.154s 开始，且录屏时以 0.5 倍速播放：

```bash
# Step 1: 进入项目目录
cd /Users/cyf/graduation_thesis

# Step 2: 安装依赖
pip install opencv-python

# Step 3: 人工标注真值
python tools/annotate_video.py \
    --video "/Users/cyf/Movies/塔吊标准节-综合预警-评估.mp4" \
    --output "gt_塔吊标准节_综合预警_评估.csv"

# Step 4: 从 ROS bag 导出预测（需先修改 export_bag_prediction.py 中的消息解析逻辑）
python tools/export_bag_prediction.py \
    --bag /path/to/your.bag \
    --topic /warning_level \
    --video-name "塔吊标准节-综合预警-评估" \
    --output prediction.csv

# Step 5: 运行指标统计（时间对齐 + 偏移 + 倍速修正）
python tools/eval_static_warning.py \
    --gt gt_塔吊标准节_综合预警_评估.csv \
    --pred prediction.csv \
    --time-offset 9.154 \
    --time-scale 0.5

# Step 6: 复制控制台输出的 LaTeX 代码，贴入论文 chap05.tex
```
